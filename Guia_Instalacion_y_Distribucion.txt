# Guía de Instalación y Distribución de Sistema de Gestión (React + Firebase)

## 1. ¿Qué es este sistema?
Este sistema es una aplicación web desarrollada en React, conectada a Firebase (Firestore, Auth y Storage), pensada para la gestión de inventario, ventas, caja y reportes. Aunque el ejemplo es para plantas, puedes adaptarlo fácilmente para otros rubros (panadería, librería, etc.) cambiando los nombres y categorías de productos.

---

## 2. Requisitos para instalar y usar en otro equipo
- Node.js instalado (https://nodejs.org/)
- Acceso a una cuenta de Firebase (plan gratuito suficiente para comenzar)
- Navegador web moderno (Chrome, Edge, Firefox)

---

## 3. Paso a paso para desplegar en otro equipo

### A. Preparar el build de producción
1. En tu equipo de desarrollo, abre una terminal en la carpeta del proyecto.
2. Ejecuta:
   ```powershell
   npm install
   npm run build
   ```
   Esto generará la carpeta `build/` con los archivos listos para producción.

### B. Copiar la carpeta build al otro equipo
1. Usa pendrive, red, Google Drive, etc. para copiar la carpeta `build/` al equipo destino.

### C. Instalar Node.js y el servidor web simple
1. Instala Node.js desde https://nodejs.org/
2. Abre una terminal y ejecuta:
   ```powershell
   npm install -g serve
   ```

### D. Servir la app localmente
1. Navega a la carpeta donde copiaste `build/`:
   ```powershell
   cd ruta\a\build
   ```
2. Ejecuta:
   ```powershell
   serve -s .
   ```
3. Abre el navegador y ve a http://localhost:5000

---

## 4. Configuración de Firebase para cada cliente
1. Crea un nuevo proyecto en https://console.firebase.google.com/
2. Habilita Firestore, Auth y Storage (si quieres subir imágenes).
3. Copia las claves del proyecto y pégalas en el archivo `.env` de tu proyecto React:
   ```env
   REACT_APP_API_KEY=...
   REACT_APP_AUTH_DOMAIN=...
   ...
   ```
4. Ejecuta `npm run build` nuevamente para que la configuración se aplique.
5. Sube la nueva carpeta `build/` al equipo o servidor del cliente.

---

## 5. Subida de imágenes y logo
- Si usas Firebase Storage, las imágenes se suben desde el formulario y se ven en el inventario automáticamente.
- Si usas imágenes locales, cópialas a la carpeta `public/img/plants/` y guarda la ruta en el producto.
- El logo puede ir en `public/imgLogo/` o la ruta que uses en tu app.

---

## 6. ¿Es gratuito?
- **Firebase:** Plan Spark es gratis para proyectos pequeños.
- **Vercel/Netlify/Firebase Hosting:** Gratis con límites generosos.
- **Node.js y serve:** Gratis.
- Si tu sistema crece mucho, puedes escalar pagando solo por lo que usas.

---

## 7. Adaptar a otros rubros
- Cambia los nombres de productos, categorías y campos en el inventario.
- Puedes usar la misma estructura para panaderías, librerías, ferreterías, etc.
- Solo debes ajustar los textos y, si quieres, los formularios de carga.

### Ejemplo: Adaptar para Panadería
1. **Cambiar textos y nombres:**
   - Busca y reemplaza la palabra "Planta" por "Pan" o "Producto" en los archivos de componentes (por ejemplo, en `src/components/` y `src/components/Movil/`).
   - Cambia los títulos de las vistas, menús y formularios.
2. **Personalizar campos:**
   - Si tu panadería necesita campos como "Tipo de pan", "Peso", "Ingredientes", edítalos en los formularios correspondientes (por ejemplo, en `PlantForm.js` o `PurchasesForm.js`).
   - Puedes agregar o quitar campos según lo que vendas.
3. **Actualizar imágenes y categorías:**
   - Coloca las imágenes de panes en `public/img/pan/` y ajusta la ruta en el inventario.
   - Cambia las categorías en la base de datos o en los formularios para reflejar los tipos de productos de tu rubro.
4. **Ajustar reportes y visualización:**
   - Modifica los títulos y columnas de los reportes para que muestren la información relevante para tu negocio.
5. **Otros rubros:**
   - Para librerías, cambia "Planta" por "Libro" y agrega campos como "Autor" o "Editorial".
   - Para ferreterías, usa "Producto" y agrega campos como "Marca" o "Medida".

> Si necesitas agregar lógica especial (por ejemplo, cálculo de stock por peso, vencimientos, etc.), consulta a tu desarrollador para modificar la lógica en los archivos de componentes y utilidades.
